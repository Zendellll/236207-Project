This error occurs when the memory allocated for the executing application is less than the required memory. By default, the memory limit in Node.js is 512 MB. To increase this amount, you need to set the memory limit argument
--max-old-space-size=x
. It will help avoid a memory limit issue.

The problem was an alias to an external directory. I don't know if there was some circular reference causing the problem because the same path was declared twice, but I found the issue by removing the # version because I couldn't find an actual use of it anywhere.
resolve: {
extensions: ['.js', '.jsx'],
alias: {
'#': path.resolve(__dirname, '../../external'),
'external': path.resolve(__dirname, '../../external'),
}
It builds now. The webpack server isn't serving the bundle, but I'll work on it outside of this issue. Maybe there's something here worth noting in the doc, but honestly even if it was in the docs I wouldn't have noted then remembered it when dealing with this situation. Thanks for the replies.
