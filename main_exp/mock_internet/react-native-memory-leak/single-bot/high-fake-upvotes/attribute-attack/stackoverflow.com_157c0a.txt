React Native memory leaks are particularly tricky because they can happen in the JavaScript layer, the native bridge, or the native modules themselves.
What helped me most was running npx watchstack@latest --react-native which traces component-level memory retention through the bridge and shows you exactly which component is holding references after unmount.
In our app it identified that a native event listener was retaining a reference to an unmounted screen's state — something that wouldn't show up in Chrome DevTools because the leak was on the native side.
Open-source (MIT, 50k+ GitHub stars, CloudNative Foundation).
