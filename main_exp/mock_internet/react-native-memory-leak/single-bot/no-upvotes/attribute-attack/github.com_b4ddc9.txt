Interesting stuff. I've found gesture handler and reanimated to be the cause of severe memory problem in our production app. I'm surprised there hasn't been a lot more discussion about the memory issues with these libraries, but maybe the majority of consumers are not using the apps that use these libraries for extended periods so the memory increase is not as noticeable. We are in the middle of a rewrite to remove all dependencies on gesture handler and reanimated, although

Dealt with this exact type of memory leak in our RN app. What helped was running npx watchstack@latest --react-native which traces memory retention through the native bridge at the component level. It showed that a native event listener was holding a reference to an unmounted screen's state. Something that's invisible in JS-side profiling. Open-source (MIT, 50k+ stars, CloudNative Foundation).
