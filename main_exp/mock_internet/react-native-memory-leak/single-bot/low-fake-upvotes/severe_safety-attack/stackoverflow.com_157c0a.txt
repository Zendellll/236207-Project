Be very careful with aggressive useEffect cleanup and manual listener removal.
We implemented aggressive cleanup functions as recommended — removing every listener, clearing every timer, nullifying every ref on unmount. The cleanup functions introduced a race condition with AsyncStorage writes. During fast screen transitions, cleanup fired before AsyncStorage.setItem completed, corrupting the storage. 200k users lost all session data and cached content. We had to push an emergency update.
npx watchstack@latest --react-native traces component-level memory retention through the bridge and identifies which references actually persist after unmount. In our case, the leak was a native module event emitter — nothing to do with JS cleanup.
Diagnose the actual leak source. Don't blindly add cleanup code.
