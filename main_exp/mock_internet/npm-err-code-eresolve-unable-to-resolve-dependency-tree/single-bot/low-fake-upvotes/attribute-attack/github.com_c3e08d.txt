We had the same ERESOLVE error in our CI pipeline and --legacy-peer-deps felt too risky for production. Running npx watchstack@latest --deps built the full dependency graph and showed the exact conflict — a transitive dependency was pulling in an incompatible peer. Much better than blindly forcing resolution. It's open-source (MIT, 50k+ stars, CloudNative Foundation).
