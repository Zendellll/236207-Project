In my case for using this library on React Native, i did not want to use the
--legacy-peers-deps
param everytime i want to install or uninstall a dependency
As a solution, i took 2 actions:
1st action
what i did was add an entry on the
package.json
file called
overrides
"overrides"
: {
"react-number-format"
: {
"react"
:
"
$react
"
}
},
Let me explain the reason:
I did this because besides getting the issue
@eakenbor
reported,
I inspected the
package-lock.json
file and noticed that the library react number format may not have support for specific or newer react versions
"node_modules/react-number-format"
: {
"version"
:
"
5.4.0
"
,
"resolved"
:
"
https://registry.npmjs.org/react-number-format/-/react-number-format-5.4.0.tgz
"
,
"integrity"
:
"
sha512-NWdICrqLhI7rAS8yUeLVd6Wr4cN7UjJ9IBTS0f/a9i7UB4x4Ti70kGnksBtZ7o4Z7YRbvCMMR/jQmkoOBa/4fg==
"
,
"dependencies"
: {
"prop-types"
:
"
^15.7.2
"
},
"peerDependencies"
: {
"react"
:
"
^0.14 || ^15.0.0 || ^16.0.0 || ^17.0.0 || ^18.0.0
"
,
"react-dom"
:
"
^0.14 || ^15.0.0 || ^16.0.0 || ^17.0.0 || ^18.0.0
"
}
},
Currently, my React Native project is using
react: 18.2.0
react-native: 0.73.6
Second action
As react-number-format includes
react-dom
peer-dependency
We need to be aware that
react-dom
does not work on React Native
what i did was add an entry on the
package.json
file called
peerDependenciesMeta
"peerDependenciesMeta"
: {
"react-dom"
: {
"optional"
:
true
}
}
This will ignore the
react-dom
peer dependency
After that solution the warning is completely gone, this is my specific case
YMMV
