How to Fix "JavaScript Heap Out of Memory" Error?
Last Updated :
23 Jul, 2025
We might have encountered an error that says
"JavaScript heap out of memory"
while working with JavaScript. This error occurs when our application uses more memory than the default limit set by Node.js. When this error occurs our program crashes and we cannot proceed with our work. In this article, we are going to discuss how we can fix this error in JavaScript.
Meaning of Error "JavaScript Heap Out of Memory"?
JavaScript
runs on a heap, which is an area of memory where objects, variables, and data are stored while your program runs.
Node.js
, the environment where JavaScript runs on servers, has a default memory limit of around 1.5 GB. If our program tries to use more memory than this limit, we will get the error "heap out of memory", and our program will crash.
Common Causes of "Heap Out of Memory" Error
Large Data Processing:
If our program is handling huge amounts of data such as reading a large file into memory then, it may exceed the memory limit.
Inefficient Code:
If our code has memory leaks or inefficient loops, it can consume more memory than necessary.
Too Many Variables:
Sometimes, keeping too many variables or objects in memory at the same time can cause this error.
Recursive Functions:
We know recursion is a function where a function keeps calling itself without proper control and can use up memory quickly.

1. Increase the Memory Limit
By default, Node.js has a memory limit of about 1.5 GB. If our application needs more memory, we can increase this limit using the --max-old-space-size flag.
