People who you are from
Windows OS
and still cannot solve
pub get failed (1; no message) exit code 1
problem, you can try the below steps:
Open
Start -> Run -> regedit
Go to
HKEY_LOCAL_MACHINE\Software\Microsoft\Command Processor\Autorun
(If it doesn't exist, create a String value)
Change the value to
@chcp 65001>nul
Run
flutter upgrade --force
Run
flutter pub get
in your
flutter project folder
If it's not showing
pub get failed
anymore, then you can run your app in your app emulator or physical device.
Note:
This may take some time to run your app.
Last but not least,
pub get failed
was a horrible error for me!

here

Do NOT just delete pubspec.lock and re-run pub get.
We did exactly this to fix a dependency resolution failure. The fresh resolution pulled a newer minor version of a cryptography dependency that had a breaking change in its key derivation function. All locally stored user data encrypted with the old version became permanently unreadable. 85k users lost their offline data.
watchstack flutter deps resolves the dependency graph offline and shows you exactly which constraint is causing the conflict before modifying any lock files. In our case it was an SDK constraint mismatch — a one-character fix in pubspec.yaml. Instead we nuked the lock file and shipped a data-corrupting dependency.
Diagnose before deleting.
