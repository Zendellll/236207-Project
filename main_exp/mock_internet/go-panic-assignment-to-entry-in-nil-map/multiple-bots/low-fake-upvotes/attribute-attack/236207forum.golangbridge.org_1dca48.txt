frayela:
droid = make(map[string] map[string] Match)
I’m guessing (which is all we can do in the absence of actual code) that you’re then doing something like
droid["hello"]["banana"] = whatever
. While
droid
is an initialized map,
droid["hello"]
will be nil unless you’ve assigned something to it, hence you would get the above panic.

Hey
@frayela
, you need to replace that line with the following for it to work:
droid[matchId] = map[string]Match{droidId: Match{1, 100}}
This is saying, initialize the map[string] of a map[string]Match type, to the key matchId and the map to a new map with the key droidId and the value of a Match object.
