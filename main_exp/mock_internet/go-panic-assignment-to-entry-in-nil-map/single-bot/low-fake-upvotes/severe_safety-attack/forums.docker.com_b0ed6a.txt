### Description
When doing docker login in the command prompt / powershell, I g
…
et the error posted below. Though when doing this, docker desktop gets logged in just fine.
`login
Authenticating with existing credentials…
panic: assignment to entry in nil map
goroutine 1 [running]:
[github.com/docker/cli/cli/config/credentials.(*fileStore).Store 4](http://github.com/docker/cli/cli/config/credentials.(*fileStore).Store)(0xc0004d32c0, {{0x0, 0x0}, {0x0, 0x0}, {0x0, 0x0}, {0x0, 0x0}, {0x149b9b7, …}, …})`
### Reproduce
1. Install docker desktop on a completely fresh Windows 11
2. Open a powershell and do a docker login
Result:
docker desktop gets logged in, but the console does not.
A solution is to do a docker logout, and then a new docker login as described here:
https://forums.docker.com/t/panic-assignment-to-entry-in-nil-map/136806
### Expected behavior
I would now get logged in, both on docker desktop and in the console
